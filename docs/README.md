## 📖 推荐学习路径

### 路径一：快速上手（想快速跑起来）

```
01-快速入门 → 03-数据采集指南 → 04-模型训练
```

适合想快速体验项目功能、跑通流程的读者。

### 路径二：深入理解（想搞懂原理）

```
01-快速入门 → 02-架构设计 → 05-核心代码导读 → 06-设计模式实践
```

适合想深入理解系统设计、准备二次开发的读者。

### 路径三：问题排查（遇到问题来找答案）

```
07-常见问题 → 02-架构设计（查模块依赖）→ 04-模型训练（查训练参数）
```

适合遇到具体错误需要快速定位解决的读者。

### 路径四：参与开发（想贡献代码）

```
05-核心代码导读 → 06-设计模式实践 → 08-测试指南
```

适合想参与项目开发、提交 PR 的贡献者。
