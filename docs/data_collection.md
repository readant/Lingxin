# 聆心手语识别系统 - 数据采集指南

| 项目 | 要求 | 说明 |

## 二、采集工具使用

### 2.1 启动采集工具

```bash
cd Lingxin
python tools/collect_data.py
```

### 2.2 输入采集信息

启动后会提示输入：
```
请输入采集人员ID（如：user001）: 
请输入目标录制数量（每个词）: 30
```

### 2.3 界面说明

采集界面包含以下信息：

| 位置 | 内容 | 说明 |
|------|------|------|
| 左上角 | 当前词汇 | 显示正在录制的手语词 |
| 右上角 | 录制进度 | 已录制次数 / 目标次数 |
| 底部 | 操作提示 | 显示可用快捷键 |
| 边框颜色 | 录制状态 | 红色=录制中，绿色=等待中 |

### 2.4 键盘控制

| 按键 | 功能 | 说明 |
|------|------|------|
| 空格键 | 开始/停止录制 | 录制时会显示帧计数 |
| N键 | 下一个词 | 切换到下一个词汇 |
| P键 | 上一个词 | 切换到上一个词汇 |
| Q键 | 显示统计后退出 | 退出前显示采集统计 |
| ESC键 | 直接退出 | 不保存统计信息 |

## 三、采集规范

### 3.1 动作规范

1. **动作标准**：按照标准手语动作执行
2. **动作连贯**：保持动作流畅，不要过于僵硬
3. **位置居中**：手部保持在画面中央
4. **角度多样**：每个词从不同角度录制（正面、侧面）

### 3.2 质量控制

系统会自动进行质量检查：
- **最短序列**：少于15帧的序列不保存（动作太短）
- **最长序列**：超过150帧的序列截取中间部分
- **手部检测**：录制时检测不到手会显示黄色警告

### 3.3 采集数量建议

| 词汇难度 | 建议样本数 | 说明 |
|----------|------------|------|
| 简单（如数字） | 20-30个 | 动作简单，样本数量可少 |
| 中等（如问候） | 30-40个 | 需覆盖不同角度 |
| 复杂（如动词） | 40-50个 | 动作变化多，需更多样本 |

## 四、数据格式

### 4.1 文件结构

```
data/
└── raw/
    └── collected/
        ├── 你好/
        │   ├── user001_001.npy
        │   ├── user001_002.npy
        │   └── ...
        ├── 谢谢/
        │   └── ...
        └── ...
```

### 4.2 数据格式说明

每个 `.npy` 文件包含：
- **数据类型**：NumPy数组
- **形状**：(帧数, 171)
- **171维特征向量组成**：
  - 左手：63维（21个关键点 × 3坐标）
  - 右手：63维（21个关键点 × 3坐标）
  - 姿态：45维（15个关键点 × 3坐标）

### 4.3 数据示例

```python
import numpy as np

# 加载一个样本
data = np.load('data/raw/collected/你好/user001_001.npy')

# 查看数据形状
print(f"序列长度：{data.shape[0]}帧")
print(f"特征维度：{data.shape[1]}维")

# 第一帧数据
first_frame = data[0]
left_hand = first_frame[:63]      # 左手关键点
right_hand = first_frame[63:126]  # 右手关键点
pose = first_frame[126:]          # 姿态关键点
```

4. **特征提取**：计算相对特征（可选）

## 六、常见问题

### 6.1 问题：检测不到手部

**现象**：画面显示黄色警告，提示"检测不到手"

**解决方案**：
1. 确保光线充足
2. 手部靠近摄像头
3. 背景不要太复杂
4. 手掌面向摄像头

### 6.2 问题：录制的序列太短

**现象**：系统提示"序列太短，至少需要15帧"

**解决方案**：
1. 按住空格键的时间更长一些
2. 动作放慢，确保录制足够帧数

### 6.3 问题：数据保存失败

**现象**：提示保存路径不存在

**解决方案**：
确保 `data/raw/collected/` 目录存在，系统会自动创建词汇子目录

### 6.4 问题：中文显示乱码

**现象**：界面上的中文显示为"???"

**解决方案**：
系统会自动查找Windows字体，确保系统安装了中文字体（如微软雅黑、黑体）
