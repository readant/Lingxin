## 一、手部检测技术

### 1.1 MediaPipe 简介

MediaPipe 是 Google 开源的计算机视觉框架，专门用于处理手部、面部、人体等关键点检测。

```
┌─────────────────────────────────────────┐
│            MediaPipe Hand Detection     │
├─────────────────────────────────────────┤
│                                         │
│   输入图像 ──→ 关键点检测 ──→ 输出      │
│                 ↓                       │
│            21个手部关键点                │
│                                         │
└─────────────────────────────────────────┘
```

### 1.2 21 个手部关键点

MediaPipe 检测的手部关键点分布如下：

```
        8   12  16  20          ← 指尖
        │    │   │   │
        7   11  15  19
        │    │   │   │
        6   10  14  18
        │    │   │   │
        5    9  13  17
         \   \   \  /
          ────────
             │
             0                 ← 手腕
```

| 关键点 | 名称 | 说明 |
|--------|------|------|
| 0 | 手腕 | 手部中心点 |
| 1-4 | 拇指 | 从根部到指尖 |
| 5-8 | 食指 | 从根部到指尖 |
| 9-12 | 中指 | 从根部到指尖 |
| 13-16 | 无名指 | 从根部到指尖 |
| 17-20 | 小指 | 从根部到指尖 |

## 二、特征提取详解

### 2.1 特征提取流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      特征提取流程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  原始关键点 ──→ 相对化处理 ──→ 手指长度 ──→ 关节角度 ──→ 特征向量│
│      │              │              │              │            │
│      ↓              ↓              ↓              ↓            │
│  21点×3维      减去手腕坐标    计算范数       arccos计算      concat │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 相对化处理

**目的**：消除手部在图像中位置的影响，让特征具有位置不变性。

```python
# 原始关键点（绝对坐标）
wrist = landmarks[0]  # 手腕坐标
relative_landmarks = landmarks - wrist  # 所有点减去手腕坐标
```

**图示**：

```
相对化前：                    相对化后：

    ● ● ●                      ● ← 原点(0,0)
      ●                        │
   ●    ●            →        ●   ●   ●
      ●                         ──────
                               （以手腕为原点）
```

### 2.3 手指长度计算

手指长度是指尖到指根的距离：

```python
fingers = [
    landmarks[5:9],   # 食指: 5→6→7→8
    landmarks[9:13], # 中指: 9→10→11→12
    landmarks[13:17],# 无名指: 13→14→15→16
    landmarks[17:21]# 小指: 17→18→19→20
]

# 计算每根手指的长度
finger_length = np.linalg.norm(finger[-1] - finger[0])
```

### 2.4 关节角度计算

**数学原理**：利用向量点积计算角度

```
    a
    ●
   / \
    │
    ● b
   / \
    │
    ● c
```

向量 $\vec{ba} = a - b$，向量 $\vec{bc} = c - b$

$$cos(\theta) = \frac{\vec{ba} \cdot \vec{bc}}{|\vec{ba}| \times |\vec{bc}|}$$

$$\theta = arccos(cos(\theta))$$

```python
def calculate_angle(a, b, c):
    v1 = a - b
    v2 = c - b

    cos_angle = np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2) + 1e-8)
    angle = np.arccos(np.clip(cos_angle, -1, 1))
    return angle
```

### 2.5 最终特征向量

最终特征向量由以下部分拼接而成：

| 部分 | 维度 | 说明 |
|------|------|------|
| 相对关键点 | 21 × 3 = 63 | 归一化后的坐标 |
| 手指长度 | 4 | 4根手指的长度 |
| 关节角度 | 4 | 4个关节的角度 |

**总维度**：63 + 4 + 4 = **71 维**

## 三、模型原理

### 3.1 SVM（支持向量机）

**适用场景**：小规模数据集，分类边界复杂

```
    ● ● ●
      ●
  ────────── ← 分类边界
    ○ ○
      ○ ○
```

**核心思想**：找到一个最优超平面，将不同类别的数据分开，且间隔最大。

### 3.2 随机森林（Random Forest）

**适用场景**：需要抗噪声、快速原型

```
决策树1: 角度>30? ─┬─Yes──→ 类A
                   └─No───→ 决策树2
                              ├─Yes──→ 类B
                              └─No───→ 类A

决策树2: 长度>50? ─┬─Yes──→ 类B
                   └─No───→ 类C

决策树3: ...
          ↓
     投票结果
```

**核心思想**：多棵决策树集成，通过投票决定最终类别。

### 3.3 MLP（多层感知机）

**适用场景**：中等规模数据，需要学习复杂模式

```
输入层(71) ──→ 隐藏层1(128) ──→ 隐藏层2(64) ──→ 输出层(num_classes)
    │              │                │
    ↓              ↓                ↓
  71个特征      ReLU激活         ReLU激活       SoftMax
```

**核心思想**：通过多层非线性变换学习复杂特征组合。

### 3.4 LSTM（长短期记忆网络）

**适用场景**：序列数据，需要捕捉时序依赖

```
┌─────────────────────────────────────────────────┐
│                   LSTM 单元                      │
├─────────────────────────────────────────────────┤
│                                                 │
│   h_{t-1} ─┐                                    │
│            ↓                                    │
│   x_t ───→ │ ←───→ 门控机制 ←───→ h_t          │
│            ↓                                    │
│   c_{t-1} ─┴──→ │ ←──→ 细胞状态 ←──→ c_t      │
│                                                 │
└─────────────────────────────────────────────────┘

遗忘门：决定丢弃哪些信息
输入门：决定更新哪些信息
输出门：决定输出哪些信息
```

**核心思想**：通过门控机制解决长期依赖问题。

### 3.5 Transformer

**适用场景**：大规模数据，需要并行处理

```
┌─────────────────────────────────────────────────┐
│              Transformer 编码器                   │
├─────────────────────────────────────────────────┤
│                                                 │
│  输入 ──→ 位置编码 ──→ 自注意力 ──→ 前馈网络    │
│           │               │               │      │
│           ↓               ↓               ↓      │
│         + Token         Multi-Head        全连接 │
│         位置信息         注意力                       │
│                                                 │
└─────────────────────────────────────────────────┘
```

**核心思想**：通过自注意力机制捕捉序列中任意位置的依赖关系。

### 4.1 性能对比

| 模型 | 参数量 | 训练时间 | 预测速度 | 准确率 |
|------|--------|----------|----------|--------|
| SVM | - | 快 | 快 | 中 |

## 五、设计模式应用

### 5.1 模板方法模式

在项目中，通过 `BaseModel` 抽象基类统一了深度学习模型的训练流程：

```python
class BaseModel(nn.Module):
    def train_model(self, train_loader, val_loader, epochs, lr):
        # 通用训练逻辑
        for epoch in epochs:
            self.train()
            for inputs, labels in train_loader:
                # 训练步骤
                pass

    @abstractmethod
    def forward(self, x):
        pass  # 子类实现
```

**好处**：消除 LSTM 和 Transformer 的重复代码（约 150 行）

### 5.2 字典映射模式

使用 `MODEL_MAP` 消除大量 if-elif 分支：

```python
MODEL_MAP = {
    'svm': {'type': 'classifier', ...},
    'lstm': {'type': 'deep_learning', ...},
    'transformer': {'type': 'deep_learning', ...}
}
```

**好处**：添加新模型只需修改配置，无需修改业务逻辑。
