## 一、环境问题

### 1.1 Python 版本问题

**问题描述**：

```
ImportError: This platform does not support PyTorch
```

**原因**：Python 版本与 PyTorch 不兼容。

**解决方案**：

```bash
# 检查 Python 版本
python --version

# 推荐使用 Python 3.8 - 3.10
# 如果版本不对，创建新环境
conda create -n lingxin python=3.9
conda activate lingxin
```

### 1.2 依赖安装失败

**问题描述**：

```
ERROR: Could not build wheels for opencv-python
```

**解决方案**：

```bash
# Windows 用户安装 Visual Studio Build Tools
# 或者尝试预编译版本
pip install opencv-python-headless

# MediaPipe 安装失败
pip install mediapipe --prefer-binary
```

### 1.3 CUDA/GPU 不可用

**问题描述**：

```python
import torch
print(torch.cuda.is_available())  # False
```

**排查步骤**：

```bash
# 1. 检查 NVIDIA 驱动
nvidia-smi

# 2. 检查 CUDA 版本
nvcc --version

# 3. 验证 PyTorch CUDA 支持
python -c "import torch; print(torch.version.cuda)"
```

**解决方案**：

```bash
# 卸载 CPU 版本
pip uninstall torch torchvision

# 安装 GPU 版本
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
```

## 二、数据问题

### 2.1 数据加载失败

**问题描述**：

```
FileNotFoundError: [Errno 2] No such file or directory: 'data/processed/X.npy'
```

**排查步骤**：

```python
import os

# 检查目录结构
print(os.listdir('data'))
print(os.listdir('data/processed'))

# 检查文件是否存在
print(os.path.exists('data/processed/X.npy'))
```

**解决方案**：

```bash
# 创建必要的目录
mkdir -p data/raw data/processed

# 检查 vocab.csv 是否有内容
head data/vocab.csv
```

### 2.2 数据格式不匹配

**问题描述**：

```
ValueError: Found input variables with inconsistent numbers of samples
```

**原因**：X 和 y 的样本数量不一致。

**排查步骤**：

```python
import numpy as np

X = np.load('data/processed/X.npy')
y = np.load('data/processed/y.npy')

print(f"X shape: {X.shape}")
print(f"y shape: {y.shape}")
print(f"X samples: {X.shape[0]}")
print(f"y samples: {y.shape[0]}")

if X.shape[0] != y.shape[0]:
    print("数据不匹配！")
```

### 2.3 数据不平衡

**问题描述**：某些类别数据过多，某些类别数据过少。

**诊断代码**：

```python
import numpy as np
from collections import Counter

y = np.load('data/processed/y.npy')
counter = Counter(y)

print("类别分布：")
for label, count in sorted(counter.items()):
    print(f"  类别 {label}: {count} 样本")

# 找出最多和最少的类别
max_count = max(counter.values())
min_count = min(counter.values())
print(f"\n不平衡比例: {max_count/min_count:.2f}:1")
```

**解决方案**：

```python
# 方案1：过采样少数类
from sklearn.utils import resample

# 方案2：欠采样多数类
# 建议使用 imbalanced-learn 库
pip install imbalanced-learn
```

## 三、模型问题

### 3.1 训练 loss 不下降

**问题描述**：训练 loss 一直是 NaN 或不下降。

**排查步骤**：

```python
# 1. 检查数据是否有 NaN
print(np.isnan(X).sum())
print(np.isnan(y).sum())

# 2. 检查数据范围是否正常
print(f"X min: {X.min()}, max: {X.max()}")

# 3. 检查学习率是否太大
# 尝试调小学习率：lr=0.0001
```

**常见原因及解决方案**：

| 原因 | 解决方案 |
|------|----------|
| 学习率过大 | 调小学习率，如 0.0001 |
| 数据未标准化 | 使用 StandardScaler |
| 数据有 NaN | 数据清洗，填充或删除 |
| 类别不平衡 | 使用类别权重或过采样 |

### 3.2 模型预测准确率为 0

**问题描述**：模型预测所有样本都是同一个类别。

**排查步骤**：

```python
# 检查预测结果分布
y_pred = model.predict(X_test)
print("预测分布:", Counter(y_pred))
print("真实分布:", Counter(y_test))

# 检查模型是否只输出一个类别
unique_preds = np.unique(y_pred)
print(f"预测类别数: {len(unique_preds)}")
```

**常见原因**：

1. **数据问题**：标签全为同一类别
2. **模型问题**：模型输出层维度与类别数不匹配
3. **训练问题**：训练轮次不够

### 3.3 深度学习模型训练很慢

**排查步骤**：

```python
import torch

print(f"GPU 可用: {torch.cuda.is_available()}")
if torch.cuda.is_available():
    print(f"GPU: {torch.cuda.get_device_name(0)}")
    print(f"显存: {torch.cuda.get_device_properties(0).total_memory / 1e9:.2f} GB}")
```

**优化方案**：

| 优化项 | 操作 |
|--------|------|
| 使用 GPU | 确保 CUDA 可用 |
| 减小 batch_size | 从 64 改为 32 或 16 |
| 减少 epochs | 先用 10 个 epoch 测试 |
| 简化模型 | 减少隐藏层大小 |

## 四、调试技巧

### 4.1 使用调试模式

```python
import debugpy

# 在代码中添加断点
debugpy.breakpoint()

# 或者使用 pdb
import pdb
pdb.set_trace()
```

### 4.2 添加日志

```python
import logging

logging.basicConfig(level=logging.DEBUG)
logger = logging.getLogger(__name__)

logger.debug(f"X shape: {X.shape}")
logger.info(f"开始训练 {model_type}")
logger.warning(f"数据不平衡: {imbalance_ratio}")
```

### 4.3 小数据集测试

```python
# 使用小数据集快速验证
X_small = X[:100]
y_small = y[:100]

# 先在少量数据上训练
model.fit(X_small, y_small)

# 验证模型能过拟合
train_acc = model.score(X_small, y_small)
print(f"训练准确率: {train_acc:.4f}")
```

### 4.4 检查梯度

```python
# PyTorch 检查梯度
for name, param in model.named_parameters():
    if param.grad is not None:
        print(f"{name}: grad norm = {param.grad.norm()}")
    else:
        print(f"{name}: 无梯度")
```

## 五、性能问题

### 5.1 内存不足 (OOM)

**问题描述**：

```
RuntimeError: CUDA out of memory
```

**解决方案**：

```python
# 1. 减小 batch_size
batch_size = 16  # 从 32 改为 16

# 2. 及时清理显存
torch.cuda.empty_cache()

# 3. 减少 DataLoader workers
num_workers = 0  # 减少数据加载进程

# 4. 使用梯度累积
accumulation_steps = 2
```

### 5.2 数据加载慢

**诊断代码**：

```python
import time

start = time.time()
for i, (X_batch, y_batch) in enumerate(train_loader):
    pass
end = time.time()

print(f"加载 {len(train_loader)} 批次耗时: {end-start:.2f}s")
print(f"平均每批次: {(end-start)/len(train_loader)*1000:.2f}ms")
```

**优化方案**：

```python
# 1. 增加 num_workers
train_loader = DataLoader(dataset, batch_size=32, num_workers=4)

# 2. 启用 pin_memory
train_loader = DataLoader(dataset, batch_size=32, pin_memory=True)

# 3. 预加载数据到内存
X_train = np.load('data/processed/X.npy')  # 一次性加载
```

## 七、自检清单

当遇到问题时，按以下顺序检查：

```
□ 1. 环境检查
   □ Python 版本正确
   □ 所有依赖安装成功
   □ GPU/CUDA 配置正确

□ 2. 数据检查
   □ 数据文件存在
   □ 数据格式正确
   □ X 和 y 样本数一致
   □ 没有 NaN 或异常值

□ 3. 模型检查
   □ 模型输出维度正确
   □ 损失函数选择正确
   □ 学习率设置合理

□ 4. 训练检查
   □ 训练 loss 在下降
   □ 验证准确率提升
   □ 没有内存溢出
```
