## 一、项目背景

### 1.1 什么是手语识别？

手语识别（Sign Language Recognition）是计算机视觉领域的一个重要研究方向，旨在让计算机能够理解和翻译手语动作。手语是聋哑人士的主要交流方式，实现自动识别对于促进聋健融合具有重要社会意义。

```
┌─────────────────────────────────────────────────────────┐
│                    手语识别流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   手语视频 ──→ 手部检测 ──→ 关键点提取 ──→ 特征向量    │
│                      ↓                                   │
│                 MediaPipe                                │
│                      ↓                                   │
│              21个手部关键点 ──────────────────────→ 特征  │
│                                                         │
│   特征向量 ──→ 模型预测 ──→ 识别结果                    │
│                      ↓                                   │
│              SVM/LSTM/Transformer                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 为什么做这个项目？

- **社会价值**：帮助聋哑人士更好地与健听人士交流
- **技术挑战**：手语动作复杂多样，识别难度高
- **学习价值**：涉及计算机视觉、机器学习、深度学习等多个领域

## 二、项目概述

### 2.1 项目名称

**聆心（Lingxin）** - 倾听心灵的声音

### 2.2 项目目标

实现对中国手语（CSL）孤立词汇的实时识别，支持多种模型架构。

### 2.3 功能特性

| 功能 | 描述 |
|------|------|
| 手部检测 | 使用 MediaPipe 检测 21 个手部关键点 |
| 特征提取 | 将关键点转换为特征向量 |
| 模型训练 | 支持 SVM、RF、MLP、LSTM、Transformer |
| 实时推理 | 支持实时手语识别 |
| API 服务 | 提供 RESTful 接口 |

## 五、数据流

### 5.1 完整数据流程

```
1️⃣ 数据采集 ──→ 2️⃣ 数据预处理 ──→ 3️⃣ 特征提取 ──→ 4️⃣ 模型训练
     │                  │                  │                  │
     ↓                  ↓                  ↓                  ↓
collect_data.py    preprocess.py    feature_extractor   train.py
     │                  │                  │                  │
     ↓                  ↓                  ↓                  ↓
.npy 文件         清洗数据          特征向量             模型文件
```

### 5.2 各阶段说明

| 阶段 | 输入 | 输出 | 关键文件 |
|------|------|------|----------|
| 数据采集 | 摄像头视频 | .npy 文件 | `collect_data.py` |
| 数据预处理 | 原始 .npy | 清洗后 .npy | `preprocess.py` |
| 特征提取 | 关键点数据 | 特征向量 | `feature_extractor.py` |
| 模型训练 | 特征向量 | 模型文件 | `train.py` |

### 6.2 模型特点对比

| 模型 | 类型 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| SVM | 传统ML | 效果好、理论完善 | 大数据慢 | 小数据集 |
| RF | 传统ML | 抗噪、不易过拟合 | 精度有限 | 快速原型 |
| MLP | 传统ML | 可学习复杂模式 | 需调参 | 中等规模 |
| LSTM | 深度学习 | 序列建模能力强 | 训练慢 | 时序数据 |
| Transformer | 深度学习 | 并行、效果好 | 需大量数据 | 大规模数据 |
