# 聆心（Lingxin）手语识别系统架构设计

## 系统架构

聆心手语识别系统采用分层架构设计，主要包含以下几个层次：

### 1. 数据层
- **原始数据**：存储在 `data/raw/` 目录，包括自采数据和MS-ASL数据集
- **预处理数据**：存储在 `data/processed/` 目录，经过特征提取和标准化处理

### 2. 核心层
- **手部检测**：基于MediaPipe的手部关键点检测
- **特征工程**：将关键点转换为有意义的特征向量
- **模型定义**：包括传统机器学习模型和深度学习模型
- **训练逻辑**：统一的训练接口
- **工具函数**：数据加载、可视化、评估指标

### 3. 应用层
- **数据采集工具**：用于采集新的手语数据
- **预处理工具**：用于数据清洗和特征提取
- **训练入口**：用于模型训练
- **评估入口**：用于模型评估
- **推理/演示**：用于实时手语识别
- **API服务**：提供RESTful API接口

## 数据流

1. **数据采集**：通过 `tools/collect_data.py` 采集手语数据，保存为.npy文件
2. **数据预处理**：通过 `tools/preprocess.py` 对原始数据进行预处理
3. **模型训练**：通过 `tools/train.py` 训练模型
4. **模型评估**：通过 `tools/evaluate.py` 评估模型性能
5. **实时推理**：通过 `tools/inference.py` 进行实时手语识别
6. **API服务**：通过 `api/app.py` 提供API接口

## 模型选择

系统支持多种模型：

1. **传统机器学习模型**：SVM、随机森林、MLP
2. **深度学习模型**：LSTM、Transformer

## 技术栈

- **编程语言**：Python
- **核心库**：NumPy、OpenCV、MediaPipe
- **机器学习**：scikit-learn、TensorFlow
- **API服务**：Flask
- **可视化**：Matplotlib、Seaborn
